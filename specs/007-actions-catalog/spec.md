# Feature Specification: 动作大全模块

**Feature Branch**: `007-actions-catalog`

**Created**: 2026-07-24

**Status**: Draft

**Input**: User description: "功能名称：动作大全模块。所属模块：packages/miniprogram。训练 Tab 下的核心内容模块，提供动作检索、筛选、详情查看能力，对接云数据库 actions 集合。页面逻辑与数据模型使用 TypeScript 编写，复用 shared 包的数据类型定义。列表：四维筛选、名称模糊搜索、封面/名称/主要部位/难度、收藏标记、「今日练什么」随机推荐。详情：基础信息、目标肌肉、步骤、发力要点、避坑指南、替代动作跳转、收藏与加入训练、底部免责声明。"

## Clarifications

### Session 2026-07-24

- Q: 详情页「加入训练」MVP 行为？ → A: 跳转至训练计划库入口（训练 Tab 内计划列表/相关页）
- Q: 「今日练什么」的推荐池？ → A: 始终从全库随机，忽略当前筛选/搜索
- Q: 收藏操作入口？ → A: 仅详情可收藏/取消；列表只展示收藏状态标记
- Q: 替代动作关系标签来源？ → A: 本版仅展示「替代动作」列表并可跳转；不强制区分降级/升级/器材标签
- Q: 每个筛选维度的选择方式？ → A: 每个维度最多选 1 个值（可清空为「不限」）

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 浏览与四维筛选动作列表 (Priority: P1)

健身新手打开训练 Tab 进入动作大全，希望按「训练部位、所用器材、难度等级、训练目标」缩小范围（每个维度最多选一个值，可清空为不限），快速看到符合自己条件的动作卡片（封面、名称、主要部位、难度标签），并识别哪些已收藏。

**Why this priority**: 四维筛选 + 列表浏览是动作大全的主入口，单独可用即可交付核心查询价值。

**Independent Test**: 仅实现列表与筛选：进入列表 → 分别/组合设置四个维度（每维单选）→ 结果仅含匹配动作；卡片字段齐全；收藏态可见（已登录且有收藏数据时）。

**Acceptance Scenarios**:

1. **Given** 用户进入动作大全列表，**When** 未设置任何筛选，**Then** 展示可浏览的动作列表，每项含封面、名称、主要部位、难度标签
2. **Given** 用户在某一维度选择单个值，**When** 应用筛选，**Then** 列表仅保留满足该条件的动作；同维不可多选
3. **Given** 用户在多个维度各选一个值，**When** 应用筛选，**Then** 列表为各条件的交集结果
4. **Given** 用户清空某一已选维度，**When** 确认清空，**Then** 该维视为「不限」，列表按剩余条件刷新
5. **Given** 用户已登录且某动作在个人收藏中，**When** 列表加载完成，**Then** 该动作卡片展示已收藏标记；未收藏则无该标记（列表不可切换收藏）
6. **Given** 某动作无封面，**When** 列表渲染该项，**Then** 显示统一缺省图且不崩溃

---

### User Story 2 - 按名称搜索动作 (Priority: P1)

用户记得动作名称（或部分关键词），希望通过搜索框模糊匹配，快速定位目标动作，并可与筛选条件叠加使用。

**Why this priority**: 与筛选并列的检索主路径；搜索可用即可显著降低找动作成本。

**Independent Test**: 输入已知动作名片段 → 结果包含该动作；清空搜索恢复列表；与某一筛选叠加后结果仍正确。

**Acceptance Scenarios**:

1. **Given** 列表中存在名称含某关键词的动作，**When** 用户输入该关键词搜索，**Then** 结果列表包含这些动作（名称模糊匹配）
2. **Given** 用户已设置筛选条件，**When** 再输入搜索关键词，**Then** 结果同时满足筛选与搜索
3. **Given** 关键词无匹配动作，**When** 搜索完成，**Then** 展示空状态说明，而非错误页或空白崩溃
4. **Given** 用户清空搜索词，**When** 确认清空，**Then** 列表恢复为当前筛选条件下的完整结果

---

### User Story 3 - 查看动作详情（新手友好） (Priority: P1)

用户从列表点进某一动作，需要看懂「练什么、怎么练、注意什么」：基础信息、主发力肌与协同肌、3–5 步图文步骤、发力/呼吸/核心要点、三条常见错误与危害说明，底部有统一免责声明。

**Why this priority**: 详情是内容消费与安全指导的核心；列表无详情则模块价值不完整。

**Independent Test**: 打开任一有完整内容的动作详情，核对各信息区块齐全、步骤 3–5 步、避坑 3 条、页底免责声明可见。

**Acceptance Scenarios**:

1. **Given** 用户从列表进入某动作详情，**When** 详情加载成功，**Then** 展示动作名称、难度、所需器材、适用场景
2. **Given** 详情已加载，**When** 用户查看目标肌肉区，**Then** 能区分主发力肌与协同肌
3. **Given** 详情已加载，**When** 用户查看动作步骤，**Then** 看到 3–5 步分点讲解，且图文可读（有图显示图，缺图用缺省图）
4. **Given** 详情已加载，**When** 用户查看发力要点，**Then** 至少覆盖发力技巧、呼吸节奏、核心收紧类提示（以内容数据为准完整展示）
5. **Given** 详情已加载，**When** 用户查看避坑指南，**Then** 看到 3 个常见错误及对应危害说明
6. **Given** 用户滑至详情页底部，**When** 阅读结束区，**Then** 看到统一免责声明（仅供参考、不构成专业健身指导、量力而行）

---

### User Story 4 - 通过替代动作形成内容联动 (Priority: P2)

用户发现当前动作不合适时，希望在详情中看到可跳转的替代动作列表，点击后进入对应动作详情，形成可浏览的内容网络。本版不强制展示降级/升级/器材等关系标签。

**Why this priority**: 提升探索与适配能力；依赖详情页存在，可在 P1 详情稳定后交付。

**Independent Test**: 打开含替代动作的详情 → 点击某一替代项 → 进入目标动作详情；返回后仍可继续浏览。

**Acceptance Scenarios**:

1. **Given** 动作配置了替代动作，**When** 用户打开详情，**Then** 展示「替代动作」列表（本版不要求降级/升级/器材分类标签）
2. **Given** 用户点击某一替代动作，**When** 跳转完成，**Then** 进入该动作详情且信息正确加载
3. **Given** 替代动作列表中某 ID 无效或已下线，**When** 渲染或点击，**Then** 不导致页面崩溃；无效项隐藏或点击时给出友好提示
4. **Given** 动作无任何替代，**When** 打开详情，**Then** 替代区不展示或展示「暂无替代」类说明，不影响其他区块

---

### User Story 5 - 收藏与取消收藏 (Priority: P2)

用户希望在动作详情把常用动作加入个人收藏，并在列表看到收藏状态标记；再次在详情操作可取消。列表不提供收藏切换，仅展示状态。收藏写入用户收藏数据，仅本人可见可改。

**Why this priority**: 宪章「我的收藏」与动作大全明确要求；可在浏览/详情可用后独立交付。

**Independent Test**: 登录后在详情收藏 → 返回列表出现标记 → 再进详情取消收藏 → 列表标记消失；未登录时给出可理解引导。

**Acceptance Scenarios**:

1. **Given** 用户已登录且未收藏某动作，**When** 在详情点击收藏，**Then** 状态变为已收藏，并写入个人收藏记录
2. **Given** 用户已收藏某动作，**When** 在详情再次点击取消收藏，**Then** 状态恢复未收藏，对应收藏记录移除或失效
3. **Given** 用户在列表页，**When** 查看已收藏/未收藏动作，**Then** 仅看到状态标记，无法在列表直接切换收藏
4. **Given** 用户未登录，**When** 在详情尝试收藏，**Then** 提示需登录/授权，且不产生他人可见的收藏数据
5. **Given** 用户已收藏若干动作，**When** 重新进入列表或详情，**Then** 收藏状态与个人收藏数据一致

---

### User Story 6 - 「今日练什么」随机推荐 (Priority: P2)

用户不知道练什么时，希望一键获得今日推荐动作，并可跳转查看详情，降低决策成本。推荐始终从全库抽样，不受当前列表筛选或搜索条件影响。

**Why this priority**: 宪章训练 Tab 明确能力；增强趣味与留存，但不阻塞列表/详情主路径。

**Independent Test**: 在已设置筛选的列表页多次点击随机推荐 → 得到有效动作并进入详情；推荐池与当前筛选无关。

**Acceptance Scenarios**:

1. **Given** 用户在列表页看到「今日练什么」入口，**When** 点击，**Then** 得到一个可用的推荐动作并可进入其详情
2. **Given** 用户已设置筛选或搜索条件，**When** 点击「今日练什么」，**Then** 仍从全库随机推荐（可推荐出当前列表未展示的动作）
3. **Given** 用户连续多次触发推荐，**When** 内容库有足够动作，**Then** 结果可变化（允许偶发重复，但不得总是同一无效结果）
4. **Given** 内容暂时不可用，**When** 触发推荐，**Then** 展示友好失败提示，不崩溃

---

### User Story 7 - 加入训练入口 (Priority: P3)

用户在详情底部希望把当前动作纳入训练使用路径，点击「加入训练」后跳转至训练计划库入口（训练 Tab 内计划列表/相关页），形成「查动作 → 选计划」闭环。

**Why this priority**: 连接「查动作」与「去训练」闭环；完整「按动作编排计划」可后续迭代，本版先交付到计划库的明确跳转。

**Independent Test**: 打开详情 → 点「加入训练」→ 进入训练计划库入口页。

**Acceptance Scenarios**:

1. **Given** 用户在动作详情页，**When** 点击「加入训练」，**Then** 跳转至训练计划库入口（训练 Tab 内计划列表/相关页）
2. **Given** 训练计划库入口暂不可用，**When** 点击「加入训练」，**Then** 仍有友好提示，不得无反馈

---

### Edge Cases

- 网络失败或云内容不可读时：列表/详情展示可重试的错误态，不白屏崩溃
- 筛选结果为空、搜索无结果：空状态文案清晰，可一键清除条件
- 封面或步骤配图缺失：统一缺省图，布局不抖动崩溃
- 步骤不足 3 步或超过 5 步：按实际内容展示；规划与验收以种子内容达标为准，UI 不因条数异常崩溃
- 避坑条目不足 3 条：展示已有条目；内容生产侧应补齐至 3 条
- 替代动作指向不存在记录：跳过或友好提示，不中断详情主流程
- 未登录访问收藏：引导登录，不写入收藏
- 快速连续点击筛选/搜索/收藏：最终展示状态与最新请求结果一致，无错乱旧数据覆盖
- 「今日练什么」在筛选生效时：仍从全库随机，不受当前筛选/搜索影响

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 系统 MUST 在小程序训练 Tab 下提供动作大全入口，列表数据来自只读的公共动作内容库
- **FR-002**: 用户 MUST 能按训练部位、所用器材、难度等级、训练目标四个维度筛选动作；每个维度最多选 1 个值（可清空为「不限」）；已选多维条件为交集
- **FR-003**: 用户 MUST 能按动作名称进行模糊搜索，且可与筛选条件叠加
- **FR-004**: 列表项 MUST 展示动作封面、名称、主要部位、难度标签，以及当前用户的收藏状态标记（在可判定时）；列表 MUST NOT 提供收藏切换操作
- **FR-005**: 系统 MUST 提供「今日练什么」随机推荐入口；推荐 MUST 始终从全库抽样且忽略当前筛选/搜索；推荐结果可进入对应动作详情
- **FR-006**: 用户 MUST 能从列表进入动作详情，并加载该动作的完整指导信息
- **FR-007**: 详情 MUST 展示基础信息：动作名称、难度、所需器材、适用场景
- **FR-008**: 详情 MUST 展示目标肌肉：主发力肌与协同肌区分可读
- **FR-009**: 详情 MUST 展示动作步骤（目标 3–5 步分点图文讲解）；缺图时使用统一缺省图
- **FR-010**: 详情 MUST 展示发力要点，覆盖发力技巧、呼吸节奏、核心收紧等提示信息
- **FR-011**: 详情 MUST 展示避坑指南：常见错误及危害说明（内容目标为 3 条）
- **FR-012**: 详情 MUST 展示「替代动作」列表（本版不强制降级/升级/器材关系标签），且用户可点击跳转至对应动作详情；关系分类标签可后续内容迭代补充
- **FR-013**: 用户 MUST 仅能在详情页收藏/取消收藏动作；收藏写入个人收藏且仅本人可读写；列表仅展示收藏状态
- **FR-014**: 详情底部 MUST 提供「加入训练」入口；点击后 MUST 跳转至训练计划库入口（训练 Tab 内计划列表/相关页）；若目标不可用 MUST 给出友好提示
- **FR-015**: 详情页底部 MUST 附带统一免责声明：仅供参考，不构成专业健身指导，请根据自身身体状况量力而行
- **FR-016**: 动作指导文案 MUST NOT 包含治疗、康复、治病、治愈等医疗承诺表述
- **FR-017**: 列表与详情在无封面/配图时 MUST 使用统一缺省图，且 MUST NOT 因空图片字段崩溃
- **FR-018**: 列表、详情、筛选与搜索的页面状态与数据契约 MUST 与共享内容模型一致，保证类型安全、无隐式宽松类型逃逸
- **FR-019**: 当内容加载失败、无结果或替代目标无效时，系统 MUST 给出用户可理解的空态/错误态/提示，并允许重试或返回

### Key Entities

- **动作（Action）**：公共训练动作内容；含名称、难度、目标、主/协同肌、器材、场景、封面、步骤、发力要点、常见错误、替代动作引用、排序权重等
- **筛选条件（ActionFilters）**：用户在四个维度上各自可选 0 或 1 个值，以及名称搜索关键词的组合；未选维度视为不限
- **动作列表项（ActionListItem）**：列表展示所需的摘要字段 + 当前用户收藏态
- **用户收藏（UserCollect）**：用户与动作（等类型）的收藏关系；归属当前用户身份
- **替代动作引用**：指向其他动作的标识；本版展示为可跳转列表，不要求关系类型字段

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 新用户在 1 分钟内可通过筛选或搜索找到并打开至少一个目标动作详情
- **SC-002**: 在内容库已就绪环境下，四维筛选与名称搜索的正确匹配率达到 100%（抽测约定用例全部通过）
- **SC-003**: 抽测不少于 10 条动作详情：基础信息、肌肉、步骤、要点、避坑、免责声明区块齐全可读；新手可用性抽访中 ≥80% 表示「能看懂怎么做」
- **SC-004**: 含替代动作的样例中，100% 可点击跳转到有效详情；无效替代不导致崩溃
- **SC-005**: 登录用户收藏/取消收藏后，重新进入列表与详情时状态一致率 100%；收藏仅本人可见
- **SC-006**: 列表首屏在常规网络下于 3 秒内可交互展示（含封面或缺省图）；图片缺失不出现破损图标占位崩溃
- **SC-007**: 「今日练什么」在内容可用时成功率 ≥95%；失败时有明确提示
- **SC-008**: 详情页底部免责声明在所有动作详情中 100% 可见

## Assumptions

- 云数据库 `actions` 公共内容已由内容底座就绪；**008 起**规模为固定 1,324 条 exercises-dataset catalog，本功能以只读消费为主，不负责内容生产。列表 MUST 使用服务端分页/过滤/搜索，不再依赖一次 `limit=100` 全量拉取
- 四维筛选字段与动作内容模型对齐：部位≈主发力肌、器材≈所需器材、难度≈难度等级、目标≈训练目标；每个维度单选（可清空）；选项枚举以共享类型/内容数据为准
- 名称模糊搜索以动作显示名为准；别名若内容中存在则可一并匹配，否则至少保证正式名称匹配
- 步骤「图文」：若单步无独立配图，可用动作封面或统一缺省图配合文字步骤，不阻塞详情交付
- 避坑「危害说明」以内容文案表达；若结构化字段为条目列表，则每条须可读出错误与风险含义
- 替代动作本版仅展示可跳转列表，不强制降级/升级/器材标签；关系分类可后续内容/模型迭代补充
- 「今日练什么」始终从全库随机，忽略当前列表的筛选与搜索条件；不因筛选结果为空而失败（内容库可用时）
- 「加入训练」MVP：点击后跳转训练计划库入口；完整「自选动作编排/按动作筛选计划」可后续迭代
- 收藏依赖微信登录身份（OpenID）；未登录不可写入；收藏切换仅在详情页，列表只读展示状态
- 页面逻辑位于小程序包；可复用的类型与纯校验逻辑复用 `packages/shared`，并通过既有同步机制供小程序使用
- Web 端不做完整动作库（宪章边界）

## Constitution Alignment _(mandatory)_

### In Scope / Out of Scope

- **In scope**: 微信小程序训练 Tab「动作大全」列表（四维筛选、搜索、收藏态、今日推荐）、详情（步骤/肌肉/要点/避坑/替代跳转列表/收藏/加入训练跳转至计划库/免责声明）；对接公共 `actions` 与用户收藏；TypeScript + shared 类型复用
- **Out of scope**: Web 完整动作库；社区/评论/UGC；运营后台 CMS；器材大全与训练计划库本体（仅允许「加入训练」跳转至计划库入口）；医疗康复指导；自建后端；本版不实现按当前动作筛选/编排计划；本版不强制替代动作关系分类标签

### Stack & Shared Logic

- **Language**: TypeScript（页面状态、列表项、详情、筛选/搜索参数均严格类型；无隐式 any）
- **Target package(s)**: `packages/miniprogram`（页面与交互）；`packages/shared`（动作/收藏等数据类型与可复用纯逻辑）
- **Shared extraction**: 动作与收藏相关类型、列表/筛选参数形状、可复用的查询参数校验 MUST 复用或扩展 shared，禁止在小程序内平行定义冲突模型
- **Backend usage**: 只读消费云数据库公共 `actions`；用户收藏写入 `user_collect`（仅本人）；图片走云存储 fileID + 本地缺省图约定；不自建服务器

### Compliance Checklist

- [x] No medical/treatment claims; disclaimer required on calculators/guidance
- [x] Privacy: minimum necessary data only; privacy policy implications noted
- [x] No extreme diet / overload training recommendations
- [x] Platform rules: WeChat category/content alignment (if miniprogram)
- [x] Brand: UI MUST consume docs/brand; calculators follow visual contract; no alternate primary palette / heavy UI kits
